Kotlin 범위 함수 실전 가이드 — SDK 개발에서 어떻게 쓰는가

let, run, apply, also, with — 각각 언제 쓰는지 헷갈린다면, SDK 개발 실무에서의 사용 패턴으로 정리합니다.

왜 범위 함수가 중요한가

Kotlin 범위 함수는 코드를 간결하게 만드는 도구이지만, 잘못 쓰면 오히려 가독성을 해칩니다. SDK 코드는 외부 개발자가 읽을 수 있으므로 일관된 사용 패턴이 중요합니다.

실전 사용 패턴

apply — 객체 초기화

Builder 패턴 대신 사용. SDK 설정 객체를 만들 때 가장 많이 씁니다.

val config = SdkConfig().apply {
    appKey = "your-app-key"
    timeout = 30_000
    isDebug = BuildConfig.DEBUG
}

let — null 안전 처리

API 응답에서 nullable 값을 처리할 때. ?.let 패턴으로 null 체크와 변환을 한 번에.

response.adData?.let { data ->
    renderAd(data)
} ?: showFallback()

also — 로깅, 디버깅

체이닝 중간에 사이드 이펙트를 넣을 때. 원본 객체를 그대로 반환합니다.

fetchOfferwall()
    .also { Log.d(TAG, "Loaded ${it.size} offers") }
    .filter { it.isActive }

run — 변환 후 결과 반환

val result = networkClient.run {
    baseUrl = API_ENDPOINT
    addHeader("Authorization", token)
    execute(request)
}

with — 같은 객체에 여러 작업

with(analyticsTracker) {
    setUserId(userId)
    setProperty("sdk_version", SDK_VERSION)
    logEvent("sdk_init")
}

판단 기준 요약

this vs it: apply, run, with는 this로 접근 (프로퍼티 설정에 적합). let, also는 it으로 접근 (다른 함수에 전달할 때 적합).

반환값: apply, also는 원본 객체 반환 (체이닝). let, run, with는 람다 결과 반환 (변환).
